# คู่มือการทำ Row-Level Security (RLS) (เตรียมสัมภาษณ์ TCS)

RLS (Row-Level Security) คือการจำกัดสิทธิ์การมองเห็นข้อมูลในระดับ "แถว (Row)" เช่น การตั้งค่าให้ "ผู้จัดการภาค West" มองเห็นเฉพาะยอดขายของภาค West เท่านั้น ส่วนผู้จัดการภาค East ก็จะมองเห็นเฉพาะของภาค East (ทั้งๆ ที่ทุกคนล็อกอินเข้ามาดู Dashboard หน้าเดียวกัน ลิงก์เดียวกัน)

หัวข้อนี้คือ **"คำถามบังคับ 100%"** สำหรับการสัมภาษณ์ตำแหน่ง Power BI Developer ระดับ Enterprise ครับ การฝึกทำในโปรเจกต์นี้จะทำให้คุณอธิบายสเต็ปการทำงานให้กรรมการฟังได้อย่างมั่นใจ

---

## 🛠️ Step 1: สร้าง Role (กำหนดสิทธิ์การมองเห็น)
1. เปิดแถบเมนู **Modeling** ด้านบนของโปรแกรม Power BI Desktop
2. คลิกปุ่ม **Manage Roles**
3. หน้าต่างตั้งค่าจะเด้งขึ้นมา ให้กดปุ่ม **Create** เพื่อสร้าง Role ใหม่
4. ตั้งชื่อ Role ใหม่นั้นว่า **`West_Manager`**
5. ในช่อง **Tables** (ตาราง) ตรงกลาง:
   - ให้คลิกเลือกตารางที่เก็บคอลัมน์ "Region" ของคุณเอาไว้ (เช่น `Fact_Sales` หรือตาราง Dimension ที่คุณแยกไว้)
6. คลิกที่ปุ่ม **`...`** (จุดสามจุด) หลังชื่อตารางนั้น > เลือก **Add filter** > แล้วเลือกคอลัมน์ **`[Region]`**
7. โปรแกรมจะสร้างสูตร DAX เริ่มต้นให้ ในกล่องข้อความด้านขวามือ ให้คุณพิมพ์สูตรแก้เป็นแบบนี้ครับ:
   ```dax
   [Region] = "West"
   ```
8. คลิกปุ่มไอคอน **ติ๊กถูก (Verify DAX)** ด้านบนของกล่องข้อความ เพื่อเช็คความถูกต้องของสูตร
9. กดปุ่ม **Save** ด้านล่างขวาเพื่อบันทึก 

*(เสร็จแล้วครับ! ตอนนี้ Power BI มี Role ของผู้จัดการภาค West อยู่ในระบบแล้ว)*

---

## 🛠️ Step 2: ทดสอบสวมรอยเป็นผู้จัดการ (View as)
ในฐานะคนสร้าง Dashboard เราสามารถจำลองมุมมองของคนอื่นได้ง่ายๆ โดยไม่ต้องสร้างแอคเคานต์จำลองเลยครับ
1. อยู่ที่แถบเมนู **Modeling** เหมือนเดิม
2. คลิกปุ่ม **View as** (ปุ่มรูปคน ที่อยู่ข้างๆ Manage Roles)
3. ติ๊กเครื่องหมายถูกที่หน้าคำว่า **`West_Manager`** แล้วกด OK
4. **สังเกตความเปลี่ยนแปลงที่หน้า Dashboard ของคุณ:**
   - จะมีแถบคาดสีเหลืองขึ้นด้านบนบอกว่า *"Now viewing as: West_Manager"*
   - กราฟโดนัท (Total Sales by Region) ที่เคยแบ่งเป็น 4 สี ตอนนี้จะ **เหลือแค่สีของภาค West 100% สีเดียว!**
   - กราฟอื่นๆ และตัวเลข Card รวมยอดขายทั้งหมด ก็จะดรอปลงมาเหลือเฉพาะยอดของภาค West เช่นกัน
5. เมื่อว้าวกับผลลัพธ์จนพอใจแล้ว ให้คลิกปุ่ม **Stop viewing** ทางขวาสุดของแถบคาดสีเหลือง Dashboard จะกลับคืนสู่สภาพเดิม (มองเห็นยอดรวมทั้งบริษัท) ครับ

---

## 💡 ทริคสำคัญเอาไว้ตอบกรรมการสัมภาษณ์ (ระดับ Pro)
ถ้ากรรมการถามว่า: *"สร้าง Role ใน Desktop เสร็จแล้ว ทำยังไงต่อ User ถึงจะใช้งานได้จริง?"*

**ให้คุณตอบตามสเต็ปนี้เลยครับ:**
> *"หลังจากผมสร้าง Role ด้วย DAX และทดสอบผ่าน View as ใน Power BI Desktop เสร็จเรียบร้อยแล้ว ผมจะต้องทำการ **Publish** Report ขึ้นไปบน **Power BI Service (Cloud)** ครับ จากนั้นผมจะเข้าไปที่ Workspace > เลือกเมนู **Security** ของตัว Dataset (Semantic Model) แล้วทำการ **พิมพ์อีเมล (หรือ AD Group) ของผู้จัดการภาค West เข้าไปผูกกับ Role `West_Manager`** ที่เราสร้างไว้ครับ หลังจากนั้น เวลาผู้จัดการคนนั้นล็อกอินเข้ามาดู Dashboard ระบบก็จะตัดข้อมูลออกให้เหลือเฉพาะของภาค West ทันทีครับ"*

ถ้าตอบแบบนี้ได้ รับรองว่ากรรมการให้ 3 ผ่านแน่นอนครับ! ลองทำตามดูนะครับ
