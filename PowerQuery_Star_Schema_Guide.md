# คู่มือการทำ Star Schema ด้วย Power Query (Superstore Dataset)

การเปลี่ยนข้อมูลแบบ Flat Table (ตารางเดียวมีทุกอย่าง) ให้กลายเป็น **Star Schema** ถือเป็นหัวใจสำคัญของการทำ Data Modeling ในระดับ Enterprise และเป็นสิ่งที่คุณต้องอธิบายให้ผู้สัมภาษณ์ฟังได้ครับ!

คอนเซปต์หลักคือ เราจะแยกตารางตาม "มิติ (Dimension)" เช่น มิติลูกค้า, มิติสินค้า และตารางที่เก็บ "ยอดขาย (Fact)" แยกไว้ตรงกลางครับ

---

## 🛠️ Step 1: สร้างตารางข้อมูลลูกค้า (`Dim_Customer`)
เราจะสกัดข้อมูลลูกค้าออกมาให้เหลือแค่รายชื่อลูกค้าแบบไม่ซ้ำกัน (Unique)

1. เปิดหน้า **Power Query Editor** (คลิกปุ่ม Transform data)
2. ไปที่แผงซ้ายมือ คลิกขวาที่ Query ชื่อ `Superstore` แล้วเลือก **Reference** (การทำ Reference คือการอ้างอิงข้อมูลต้นฉบับ หากต้นฉบับอัปเดต ตารางนี้จะอัปเดตตาม)
3. เปลี่ยนชื่อ Query ใหม่ทางขวามือเป็น **`Dim_Customer`**
4. กดปุ่ม `Ctrl` ค้างไว้ที่คีย์บอร์ด แล้วคลิกเลือกหัวคอลัมน์ 3 อันนี้: 
   - `Customer ID`
   - `Customer Name`
   - `Segment`
5. คลิกขวาที่หัวคอลัมน์ไหนก็ได้ที่เลือกไว้ แล้วกด **Remove Other Columns** (คอลัมน์อื่นจะหายไปหมด)
6. **ขั้นตอนนี้สำคัญมาก:** คลิกขวาที่หัวคอลัมน์ `Customer ID` แล้วเลือก **Remove Duplicates** (เพื่อลบรายชื่อลูกค้าที่ซ้ำกันออก ให้เหลือลูกค้า 1 คนต่อ 1 บรรทัด)

---

## 🛠️ Step 2: สร้างตารางข้อมูลสินค้า (`Dim_Product`)
ทำคล้ายๆ กับลูกค้า แต่เปลี่ยนเป็นข้อมูลสินค้าแทน

1. ไปที่แผงซ้ายมือ คลิกขวาที่ Query ชื่อ `Superstore` (ต้นฉบับ) แล้วเลือก **Reference** อีกครั้ง
2. เปลี่ยนชื่อ Query ใหม่เป็น **`Dim_Product`**
3. กด `Ctrl` เลือกคอลัมน์เหล่านี้:
   - `Product ID`
   - `Category`
   - `Sub-Category`
   - `Product Name`
4. คลิกขวา เลือก **Remove Other Columns**
5. คลิกขวาที่หัวคอลัมน์ `Product ID` แล้วเลือก **Remove Duplicates** (เพื่อให้รหัสสินค้าไม่ซ้ำกัน)

---

## 🛠️ Step 3: ทำความสะอาดตารางยอดขายหลัก (`Fact_Sales`)
ตอนนี้เรามีมิติลูกค้าและสินค้าแยกออกไปแล้ว เราไม่จำเป็นต้องเก็บชื่อลูกค้าและชื่อสินค้าไว้ในตารางหลักอีกต่อไป (ลดขนาดไฟล์ให้เบาลง)

1. คลิกขวาที่ `Superstore` (ต้นฉบับ) แล้วเลือก **Reference** อีกครั้ง (หรือจะ Duplicate ก็ได้)
2. เปลี่ยนชื่อเป็น **`Fact_Sales`**
3. คราวนี้ให้ **ลบ (Remove)** คอลัมน์ที่เป็นรายละเอียดของ Dimension ทิ้งไป เพราะเรามี ID เอาไว้เชื่อมความสัมพันธ์แล้ว ให้ลบคอลัมน์ต่อไปนี้:
   - `Customer Name`
   - `Segment`
   - `Category`
   - `Sub-Category`
   - `Product Name`
   *(เราจะเหลือแค่ Customer ID และ Product ID เอาไว้เป็นตัวเชื่อมครับ)*

*(หมายเหตุ: เพื่อไม่ให้งงตอนดึงข้อมูลมาใช้ แนะนำให้คลิกขวาที่ Query `Superstore` อันแรกสุด เลิกติ๊กคำว่า **Enable load** ตัวหนังสือมันจะเป็นตัวเอียงๆ แปลว่าตารางนี้ใช้เป็นทางผ่านเฉยๆ จะไม่โหลดยัดเข้าไปให้หนักหน้า Report ครับ)*

---

## 🛠️ Step 4: ผูกความสัมพันธ์ (Data Modeling)
1. กดปุ่ม **Close & Apply** ที่มุมซ้ายบน เพื่อเซฟและกลับมาที่หน้า Power BI Desktop
2. ไปที่แท็บ **Model view** (ไอคอนรูปความสัมพันธ์ 3 กล่องด้านซ้าย)
3. คุณจะเห็นตารางใหม่โผล่ขึ้นมา ให้คุณลากเส้นเชื่อม (Relationship) ดังนี้:
   - ลาก `Customer ID` จากตาราง **Dim_Customer** ไปหา `Customer ID` ในตาราง **Fact_Sales**
   - ลาก `Product ID` จากตาราง **Dim_Product** ไปหา `Product ID` ในตาราง **Fact_Sales**
   - (อย่าลืมลาก `Date` จาก **Dim_Date_PQ** ไปหา `Order Date` ใน **Fact_Sales** ด้วยนะครับ)

🎉 **เสร็จเรียบร้อย!** โมเดลของคุณจะกลายเป็นรูปดาว (Star Schema) ที่ตรงกลางคือตารางยอดขาย (Fact) และถูกล้อมรอบด้วยตารางมิติ (Dimension) อย่างถูกต้องตามหลักวิศวกรรมข้อมูลครับ! ลองทำดูแล้วทักมาบอกผมได้เลยนะครับว่าเจอปัญหาตรงไหนไหม
